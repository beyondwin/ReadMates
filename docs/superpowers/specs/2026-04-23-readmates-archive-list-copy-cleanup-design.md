# ReadMates Archive List Copy Cleanup Design

작성일: 2026-04-23
상태: VALIDATED DESIGN SPEC
문서 목적: `/app/archive`의 세션 리스트와 피드백 문서 리스트에서 반복되는 피드백 상태 문구와 회차 표기를 정리해, 데스크톱과 모바일 모두에서 더 빠르게 읽히는 기록 목록으로 만든다.

## 1. 문제 정의

현재 아카이브 세션 리스트는 `참석`, `공개 기록`, `피드백 문서 있음/없음/잠김`을 모두 배지로 표시한다. 특히 `피드백 문서 있음`, `피드백 문서 없음`, `피드백 잠김`은 탭 이름인 `피드백 문서`와 같은 단어를 반복해 행이 무겁게 보인다.

피드백 문서 탭도 `No.01 · 독서모임 1차 피드백`처럼 같은 회차 정보를 `No.01`과 `1차`로 두 번 말한다. 문서가 하나의 회차 산출물임은 이미 탭과 액션 맥락에서 드러나므로, 리스트 행에서는 책 제목과 날짜, 등록일을 우선해야 한다.

## 2. 목표

- 데스크톱 세션 리스트의 상태 배지를 짧고 일관된 문구로 정리한다.
- 모바일 세션 카드도 같은 의미 체계를 유지하되 더 짧은 문구를 사용한다.
- 피드백 문서 탭에서 회차 숫자 반복을 없앤다.
- 피드백 문서 탭의 주 제목은 책 제목으로 두고, 회차/모임일/등록일은 메타 정보로 합친다.
- 기존 링크 대상, 접근 권한, 스크롤 복귀, 탭 동작은 바꾸지 않는다.
- 접근성 라벨은 화면 문구보다 조금 더 설명적으로 유지해 아이콘 버튼의 의미를 잃지 않게 한다.

## 3. 비목표

- 서버 API, 데이터 모델, 권한 정책은 바꾸지 않는다.
- 새 탭이나 필터를 추가하지 않는다.
- 피드백 문서 잠김 상태를 피드백 문서 탭에 새로 노출하지 않는다. 현재 탭은 열람 가능한 문서 목록이라는 의미를 유지한다.
- 책 표지, 색상 토큰, 전체 아카이브 레이아웃을 재설계하지 않는다.

## 4. 확정된 방향

선택한 방향은 `정보는 유지하되 문구와 계층만 정리`다.

세션 리스트에서는 피드백 상태 정보를 제거하지 않는다. 사용자가 세션 목록을 훑을 때 어느 회차에 문서가 있는지 바로 알 수 있기 때문이다. 대신 `피드백 문서 있음/잠김/없음`을 `피드백 있음/피드백 잠김/피드백 없음`으로 줄인다. `문서 가능`처럼 기능 조건만 말하는 문구는 한국어 UI 배지로 어색하므로 쓰지 않는다.

피드백 문서 탭에서는 문서 제목을 그대로 노출하지 않고 책 제목을 행의 주 제목으로 사용한다. 기존 `독서모임 1차 피드백`은 이미 탭과 문서 상세에서 충분히 설명되므로 리스트에서는 반복을 줄인다.

## 5. Desktop Design

### 5.1 세션 탭

데스크톱 세션 행의 구조는 유지한다.

- 왼쪽: `SessionIdentity`와 모임 날짜
- 가운데: 책 표지, 책 제목, 저자
- 오른쪽: 참석/공개/문서 상태 배지
- 끝: 세션 이동 액션

문구만 다음처럼 정리한다.

- `참석 6/6` 유지
- `공개 기록` 유지
- `비공개 기록` 유지
- `피드백 문서 있음` -> `피드백 있음`
- `피드백 잠김` 유지
- `피드백 문서 없음` -> `피드백 없음`

배지 class와 상태 색상은 기존 `feedbackArchiveBadgeClass` 결과를 유지한다.

### 5.2 피드백 문서 탭

데스크톱 피드백 문서 행은 현재 아이콘 전용 액션 구조를 유지한다.

주 제목:

```text
팩트풀니스
```

메타:

```text
No.01 · 2025.11.26 · 2026.04.20 등록
```

문서 읽기와 PDF 저장 버튼의 accessible label은 화면에 보이지 않는 설명이므로 다음처럼 유지하되, 중복 title을 개선한 형식으로 바꾼다.

```text
No.01 팩트풀니스 피드백 문서 읽기
No.01 팩트풀니스 피드백 문서 PDF로 저장
```

## 6. Mobile Design

### 6.1 세션 탭

모바일 세션 카드는 한 행 폭이 좁으므로 배지를 더 짧게 표시한다.

- `참석 6/6` -> `6/6 참석`
- `공개 기록` -> `공개`
- `비공개 기록` -> `비공개`
- `피드백 문서 있음` -> `피드백 있음`
- `피드백 잠김` 유지
- `피드백 문서 없음` -> `피드백 없음`

데스크톱과 모바일이 같은 helper를 공유해야 한다면 `variant` 인자를 둔다. 문구만 다르고 상태 판단은 하나의 함수에 둔다.

### 6.2 피드백 문서 탭

모바일 피드백 문서 행은 기존 `m-list-row` 구조를 유지한다. 제목과 메타는 버튼을 밀지 않게 분리한다.

주 제목:

```text
팩트풀니스
```

메타 1:

```text
No.01 · 2025.11.26
```

메타 2:

```text
2026.04.20 등록
```

아이콘 버튼은 그대로 오른쪽에 둔다.

## 7. Implementation Boundaries

주요 변경 위치:

- `front/features/archive/model/archive-model.ts`
- `front/features/archive/ui/archive-page.tsx`
- `front/tests/unit/archive-page.test.tsx`
- 필요한 경우 `front/tests/e2e/responsive-navigation-chrome.spec.ts`

권장 helper:

- `feedbackArchiveLabel(feedbackDocument, variant?)`
- `publicationLabel(published, variant?)`
- `feedbackReportActionLabel(report, action)`
- `feedbackReportMeta(report)`

기존 API contract와 라우팅 helper는 유지한다.

## 8. Testing

단위 테스트는 다음을 검증한다.

- 데스크톱 세션 리스트에 `피드백 있음`, `피드백 없음`, `피드백 잠김`이 표시된다.
- 모바일 세션 리스트에 `6/6 참석`, `공개`, `피드백 있음`처럼 짧은 문구가 표시된다.
- 피드백 문서 탭에서 `독서모임 1차 피드백`이 리스트 본문에 반복 노출되지 않는다.
- 피드백 문서 행의 링크 href, return state, aria-label이 유지된다.
- 빈 상태 문구는 기존처럼 `아직 열람 가능한 피드백 문서가 없습니다.`를 사용한다.

필요하면 현재 열려 있는 `/app/archive`를 브라우저에서 확인해 데스크톱과 모바일 폭 모두에서 버튼과 텍스트가 겹치지 않는지 확인한다.

## 9. Acceptance Criteria

- `/app/archive` 세션 탭에서 `피드백 문서 있음`, `피드백 문서 없음`, `피드백 잠김`이 더 이상 카드/행 배지로 보이지 않는다.
- 세션 탭의 피드백 상태는 `피드백 있음`, `피드백 없음`, `피드백 잠김` 중 하나로 보인다.
- 피드백 문서 탭 리스트에서 `No.01`과 `1차`가 같은 행에 반복되지 않는다.
- 데스크톱과 모바일 모두에서 주 제목, 메타, 액션 버튼이 서로 밀리거나 겹치지 않는다.
- 기존 세션 링크, 피드백 읽기 링크, PDF 링크, 탭 전환 동작은 유지된다.
