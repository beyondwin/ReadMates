# 공개 기록 메타데이터 반응형 개선 설계

작성일: 2026-07-11
상태: APPROVED DESIGN

## 문제

`/records`의 각 공개 기록 행은 책 표지 옆 본문 첫 줄에 회차·날짜와 상태·기록 건수를 두 열로 배치한다. 현재 모바일 규칙도 `minmax(0, 1fr) max-content`와 `nowrap`을 유지해, 390px 화면에서 날짜와 상태·건수 영역이 서로 약 61px 겹친다. 날짜가 사라진 것처럼 보이는 현상은 데이터 문제가 아니라 고정 폭 콘텐츠 두 묶음을 한 줄에 강제로 유지한 반응형 레이아웃 문제다.

## 목표

- 320px부터 넓은 데스크톱까지 회차·날짜, 상태 배지, 기록 건수가 서로 겹치거나 잘리지 않는다.
- 모바일에서 날짜를 가장 먼저 온전히 읽을 수 있다.
- 상태와 기록 건수를 숨기거나 글자를 과도하게 줄이지 않는다.
- 기존 책 표지, 제목, 저자, 요약, 링크 구조와 데스크톱 정보 위계를 보존한다.

## 선택한 설계

기존 행의 바깥쪽 `책 표지 | 본문` 구조는 유지한다. 본문 메타데이터만 콘텐츠가 들어갈 수 있는 폭을 기준으로 전환한다.

- 넓은 화면: 회차·날짜와 상태·건수를 같은 줄의 두 열로 유지한다.
- 좁은 화면: 본문 메타데이터 그리드를 한 열로 바꾸고 회차·날짜를 첫 줄, 상태·건수를 다음 줄에 둔다.
- 좁은 화면의 상태·건수는 왼쪽 정렬하고 필요한 경우 자연스럽게 줄바꿈한다.
- 제목, 저자, 요약은 그 아래에서 기존 순서를 유지한다.
- 전환 지점은 특정 기기 이름이 아니라 실제 콘텐츠가 겹치기 시작하는 현재 공개 페이지의 콘텐츠 폭을 기준으로 정한다.

## 검토한 대안

### 글자 크기 축소

한 줄 배치를 더 오래 유지할 수 있지만 320~390px에서 공개 기록의 가독성을 희생하고 긴 번역 문구에서 다시 깨질 수 있어 제외한다.

### 기록 건수 숨김

겹침은 사라지지만 공개 기록의 밀도 정보를 모바일 사용자에게만 제거하므로 제외한다.

### 책 표지를 본문 위로 이동

가장 좁은 화면에는 여유가 생기지만 행 높이와 스캔 리듬이 크게 달라지는 범위 확장이라 이번 결함 수정에서는 제외한다.

## 테스트 전략

Playwright E2E에서 실제 렌더링된 경계 상자를 비교한다.

- 320px, 375px 또는 390px: 날짜와 상태·건수가 서로 다른 행이며 수평·수직으로 겹치지 않는다.
- 768px과 1024px: 콘텐츠 전환 경계에서 메타데이터가 행 밖으로 넘치지 않는다.
- 1366px: 기존 두 열 정보 위계가 유지되고 모든 메타데이터가 행 안에 있다.
- 각 크기에서 행의 `scrollWidth`가 `clientWidth`를 넘지 않는지 확인한다.
- 긴 날짜/상태 텍스트가 주어져도 레이아웃이 겹치지 않는 회귀 조건을 컴포넌트 또는 브라우저 테스트에 포함한다.

## 변경 범위

- `front/src/styles/globals.css`: 공개 기록 행의 콘텐츠 기반 반응형 규칙
- `front/tests/e2e/responsive-navigation-chrome.spec.ts`: 대표 화면 폭의 비겹침 회귀 검증
- 필요할 경우 `front/features/public/ui/public-records-page.ct.tsx`: 긴 콘텐츠 시각 검증 fixture

API, 데이터 모델, 라우트, 공개 카피, 서버 코드는 변경하지 않는다.

## 완료 조건

- 대표 화면 폭 모두에서 날짜·상태·건수의 겹침과 가로 오버플로가 없다.
- 모바일에서 날짜가 완전하게 보이고 상태·건수도 보존된다.
- 프론트 lint, unit test, build와 대상 E2E가 통과한다.
- 실제 브라우저에서 모바일·태블릿·데스크톱 화면을 다시 확인한다.
