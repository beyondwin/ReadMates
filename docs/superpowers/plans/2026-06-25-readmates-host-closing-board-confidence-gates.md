# ReadMates Host Closing Board Confidence Gates Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add repeatable server query-budget, MySQL plan, frontend component visual, and documentation gates for the host session closing board.

**Architecture:** Keep the existing `sessionclosing` read-side production behavior. Add a bounded request-level query count test, share the closing-status SQL with an EXPLAIN guard to avoid test/query drift, and add a Playwright component screenshot baseline for `SessionClosingBoard`. Document the exact confidence lane in `docs/development/test-guide.md`.

**Tech Stack:** Kotlin/Spring Boot, MockMvc, Testcontainers MySQL, JDBC EXPLAIN assertions, React 19, Playwright Component Testing, Markdown docs.

## Global Constraints

- Follow `docs/agents/server.md` for server test and adapter changes.
- Follow `docs/agents/front.md` and `docs/agents/design.md` for frontend UI/visual test changes.
- Follow `docs/agents/docs.md` for `docs/development/test-guide.md`.
- Do not change the host closing status API response schema.
- Do not add a DB migration in this plan.
- Do not add platform-admin session-closing mutations.
- Do not add notification event types.
- Do not add real member data, raw email, private domains, deployment state, local absolute paths, OCIDs, secrets, token-shaped examples, raw feedback body, provider raw error, or transcript text.
- Use only public-safe placeholder UUIDs, existing dev-seed safe slugs, and public-safe Korean copy in tests and docs.
- Docker-generated CT baselines are the only committed visual baseline source. Do not commit macOS-local PNG updates.

```kotlin
internal object SessionClosingStatusSql {
    val CLOSING_BASE =
        """
        select
          sessions.id,
          sessions.number,
          sessions.book_title,
          sessions.session_date,
          sessions.state,
          sessions.visibility,
          public_session_publications.public_summary,
          public_session_publications.is_public,
          public_session_publications.published_at,
          exists (
            select 1
            from session_feedback_documents
            where session_feedback_documents.club_id = sessions.club_id
              and session_feedback_documents.session_id = sessions.id
          ) as feedback_uploaded,
          (
            select count(*)
            from highlights
            where highlights.club_id = sessions.club_id
              and highlights.session_id = sessions.id
          ) as highlight_count,
          (
            select count(*)
            from one_line_reviews
            where one_line_reviews.club_id = sessions.club_id
              and one_line_reviews.session_id = sessions.id
          ) as one_liner_count
        from sessions
        left join public_session_publications
          on public_session_publications.club_id = sessions.club_id
         and public_session_publications.session_id = sessions.id
        where sessions.id = ?
          and sessions.club_id = ?
        """.trimIndent()

    val LATEST_NOTIFICATION_EVENT =
        """
        select event_type, status, created_at
        from notification_event_outbox
        where club_id = ?
          and aggregate_id = ?
          and event_type in ('FEEDBACK_DOCUMENT_PUBLISHED', 'NEXT_BOOK_PUBLISHED')
        order by created_at desc, id desc
        limit 1
        """.trimIndent()
}
```

        val basePlan =
            jdbcTemplate.explain(
                SessionClosingStatusSql.CLOSING_BASE,
                "00000000-0000-0000-0000-000000000306",

            )

        basePlan.assertUsesIndexFor("public_session_publications", "host closing public publication join")
