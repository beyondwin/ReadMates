# ReadMates v1.7.0 후속 하드닝 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** v1.7.0 이후 35개 커밋 검토에서 확인한 잔여 결함 7건을 ReadMates 기존 아키텍처와 공개 저장소 안전 기준 안에서 닫는다. 사용자 가시 동작은 바뀌지 않고, 운영자/보안 가시성 결함과 코드 위생 부채를 정리한다.

**Architecture:** BFF 트러스트 판단은 서버 security filter 안에서만, HTTP 상태 매핑은 inbound web adapter에서만, 공개 release 안전은 후보 생성과 검사 두 곳에서 같은 정책을 따른다. 운영 스크립트는 read-only 진단과 mutating deploy를 구분된 종료 경로로 보고한다.

**Tech Stack:** Kotlin, Spring Boot, JUnit 5, ArchUnit, Bash, GitHub Actions, Trivy(action), Docker Buildx, Cloudflare Pages Functions BFF, Vite/React frontend checks.

- [ ] **Step 3: 죽은 코드 제거와 timing-uniform 리팩터**

`BffSecretFilter.kt`에서 `matchesAny` 정의를 삭제한다. `aliasFor`를 다음과 같이 교체한다.

```kotlin
private fun findMatchingSecretIndex(provided: String): Int {
    val providedBytes = provided.toByteArray(StandardCharsets.UTF_8)
    var matchedIndex = -1
    secrets.forEachIndexed { idx, candidate ->
        val candidateBytes = candidate.toByteArray(StandardCharsets.UTF_8)
        if (MessageDigest.isEqual(providedBytes, candidateBytes)) {
            matchedIndex = idx
        }
    }
    return matchedIndex
}

internal fun aliasFor(provided: String): String? =
    when (val idx = findMatchingSecretIndex(provided)) {
        -1 -> null
        0 -> "primary"
        1 -> "secondary"
        else -> "index_$idx"
    }
```
