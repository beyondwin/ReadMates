# ReadMates Mobile Role Screens Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix the standalone mobile prototype so guest, member, and host modes have distinct mobile navigation and role-appropriate home, session, and archive actions.

**Architecture:** Edit the source design modules under `design/src/mobile` and `design/styles/mobile.css`, then regenerate `design/standalone/mobile.html` with the existing standalone build script. Keep this as a design-prototype-only change and do not touch `front/` application code in this pass.

**Tech Stack:** Static React 18 JSX embedded through Babel Standalone, generated standalone HTML, CSS tokens in `design/styles/mobile.css`, Node build and verification scripts.

---

## Scope And Files

**Modify:**
- `design/src/mobile/shell.jsx`: role-specific bottom tab configuration and active-state handling for host edit routes.
- `design/src/mobile/pages-home.jsx`: member home hierarchy, member preparation actions, and mobile-first summary sections.
- `design/src/mobile/pages-session.jsx`: mobile session layout, role-aware member vs host session content, sticky segment labels.
- `design/src/mobile/pages-archive-me.jsx`: icon-only archive actions for reading and PDF/download, with accessible labels.
- `design/src/mobile/pages-host.jsx`: host landing shortcuts for the new host tab composition.
- `design/styles/mobile.css`: small reusable mobile utilities for icon buttons, action tiles, and timeline rows.
- `design/읽는사이 모바일.html`: app-entry source if context needs `activeTab` or `subRoute` passed into `MobileCtx`.
- `design/standalone/mobile.html`: generated by `node design/standalone/build.mjs`; do not hand-edit after source changes.

**Do not modify:**
- `front/**`: current unrelated app changes are already present in the worktree and should remain untouched.
- `design/standalone/index.html`: desktop prototype is out of scope.

---

## Intended Product Behavior

Guest:
- No bottom tab bar.
- Public landing and login remain the only guest states.
- Role switcher remains available for prototype preview.

Member:
- Bottom tabs: `홈`, `세션`, `아카이브`, `마이`.
- Home focuses on "내가 지금 해야 할 일": next session, reading progress, question count, RSVP, Zoom link, member activity, and archive shortcuts.
- Session is mobile-first: compact hero, sticky segment control, large touch targets, single-column content, and no desktop-style dense row layout.

Host:
- Bottom tabs differ from member: `운영`, `편집`, `아카이브`, `마이`.
- `운영` opens `MHostHomePage`.
- `편집` opens `MHostEditPage` through `goto('host-edit')` and shows the edit tab as active.
- Host can still inspect session context, but the bottom tab composition should not look like member mode.

Archive:
- In report/session action rows, visible action text such as `읽기` and `PDF로 저장` must be replaced by icon-only buttons.
- Icon-only buttons must include `aria-label` and `title` so accessibility and hover meaning are preserved.

### Task 2: Split Bottom Tabs By Role

**Files:**

- [x] **Step 1: Add role-specific tab definitions in `TabBar`**

Replace the current single `tabs` array in `design/src/mobile/shell.jsx` with this logic:

```jsx
function TabBar() {
  const { tab, setTab, role, activeTab } = useMobile();
  const memberTabs = [
    { id: 'home', label: '홈', icon: 'home' },
    { id: 'session', label: '세션', icon: 'session' },
    { id: 'archive', label: '아카이브', icon: 'archive' },
    { id: 'me', label: '마이', icon: 'me' },
  ];
  const hostTabs = [
    { id: 'home', label: '운영', icon: 'host' },
    { id: 'host-edit', label: '편집', icon: 'edit' },
    { id: 'archive', label: '아카이브', icon: 'archive' },
    { id: 'me', label: '마이', icon: 'me' },
  ];
  const tabs = role === 'host' ? hostTabs : memberTabs;
  const current = activeTab || tab;

  return React.createElement('nav', { className: 'm-tabbar' },
    ...tabs.map(t => React.createElement('button', {
      key: t.id,
      className: 'm-tab',
      'aria-current': current === t.id ? 'page' : undefined,
      onClick: () => setTab(t.id),
    },
      React.createElement(Icon, { name: t.icon, size: 20 }),
      React.createElement('span', { className: 'm-tab-label' }, t.label),
    ))
  );
}
```
