# ReadMates Route UI Visual Regression Gate Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Expand ReadMates' existing Playwright component screenshot baselines from shared primitives and one host board state into route-critical host, admin, and public UI gates.

**Architecture:** Keep production route, API, server, auth, and BFF behavior unchanged. Add props-driven Playwright CT fixtures beside the existing feature UI files, generate canonical Docker baselines under `front/__screenshots__`, and document when route-critical UI should use committed CT baselines instead of E2E screenshot artifacts.

**Tech Stack:** React 19, Vite, Playwright Component Testing, `@playwright/experimental-ct-react`, TypeScript, Markdown docs, public release candidate scripts.

## Global Constraints

- Follow `docs/agents/front.md` for all `front/` changes.
- Follow `docs/agents/design.md` for visual regression and UI-copy-sensitive changes.
- Follow `docs/agents/docs.md` for `docs/development/test-guide.md`.
- Do not change server API contracts, DB migrations, auth/BFF, OAuth, or deploy workflow behavior.
- Do not add a visual regression SaaS, Lighthouse CI gate, or route-wide committed E2E screenshots.
- Keep all CT fixtures public-safe: no real member data, private email/domain, deployment state, local absolute paths, OCIDs, secrets, token-shaped values, raw feedback body, provider raw error, or transcript text.
- Use deterministic Korean copy, sample club/session/book values, and placeholder/dev-seed-style UUIDs only.
- Docker `mcr.microsoft.com/playwright:v1.60.0-jammy` is the canonical baseline generation path. Do not commit macOS-local PNG updates.
- `front/__screenshots__` baselines remain committed source artifacts but must stay excluded from public release candidates.
